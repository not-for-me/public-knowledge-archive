# 02. Intelligent Systems: A Hybrid Approach

## 챕터 개요 (3줄 요약)

- 지능형 시스템은 지식 표현(knowledge representation)과 추론(reasoning)이라는 두 핵심 구성요소로 분해되며, KG와 LLM이 이 둘을 보완적으로 담당한다.
- 시스템은 사용자를 대체하는 자율 시스템(autonomous system)과 사용자를 보조하는 지능형 조언 시스템(IAS, Intelligent Advisor System)으로 나뉘며, 본서는 IAS에 집중한다.
- 가용 데이터에서 출발하는 상향식(bottom-up) 방식 대신, 비즈니스 목표에서 출발하는 목적 주도(purpose-driven) KG 구축이 성공의 핵심이다.

---

## 1. What Is Intelligence?

> 지능은 지식을 획득·적용하는 능력이며, 지능형 에이전트는 이를 지식 표현과 추론이라는 두 요소로 분해할 수 있다.

- 지능은 경험에서 학습하고 문제를 풀며 환경과 상호작용하는 능력으로, 시스템 설계를 위해 이를 작업·구성요소로 분해해야 한다.
- 지식 표현은 AI가 도메인 정보를 구조화·인코딩·전달하는 "언어"이며, 표현 방식 선택이 시스템 능력과 한계를 크게 좌우한다.
- 추론은 정보 분석·규칙 적용·결론 도출의 인지 과정으로 연역(deductive)·귀납(inductive) 등 여러 유형이 있다.
- 표현의 표현력(expressiveness)과 처리 효율성 사이에는 트레이드오프가 존재하며, 효과적 표현은 도메인·작업에 의존한다.
- KG는 지식을 효율적으로 표현하고 정확한 추론의 기반을 놓는 핵심 데이터 구조이며, LLM과 결합하면 데이터→지식 변환과 답변 제공을 향상시킨다.

## 2. Designing an Intelligent System

> Geoff Hulten의 정의에 따르면 지능형 시스템은 사용자를 AI/ML에 연결하여 의미 있는 목표를 달성하고, 사용자 상호작용을 관찰하며 시간에 따라 진화·개선된다.

- 지능형 시스템의 주 목적은 사용자를 대체하는 것이 아니라 의사결정 능력을 향상시켜 복잡한 작업을 보조하는 것이다(예: 자율 의료 진단 시스템은 의사를 보조).
- 시스템은 사용자 피드백·맥락 정보로부터 학습하며 진화하는 지식 베이스(knowledge base)를 지속 유지해야 한다.

### 2.2.2 Categories of Intelligent Systems

- 자율 시스템(autonomous): 완전 자동화, 실시간 의사결정, 적응성을 특징으로 사용자를 대체한다(예: 자율주행차).
- 조언 시스템(IAS): 의사결정 지원, 맥락 인식(context awareness), 사용자 상호작용을 특징으로 실행은 하지 않고 정보·추천을 제공한다.
- IAS 사례: 법집행의 예측 치안(predictive policing), 금융의 이상거래·사기 탐지, 의료의 진단·치료 추천 — 모두 최종 결정은 인간이 한다.

### 2.2.3 Characteristics of an Intelligent System

- 네 가지 핵심 특성: 의미 있는 목표(meaningful objective), 지능형 경험(intelligent experience), 지식 생성·갱신(knowledge creation/update), 오케스트레이션(orchestration).
- 설계는 자율 조언 시스템에 집중하고, 일반 지식이 아닌 논문·온톨로지·구조화 소스 기반의 도메인 지식을 활용하며, 결과 피드백으로 지식을 확장한다.

## 3. Knowledge Acquisition and Representation

> 지식 획득은 원시 데이터를 추론에 적합한 구조적 지식 표현으로 변환하는 과정이며, KG와 LLM이 크게 다른 지점이다.

- KG는 원시·구조화 데이터를 엔티티=노드, 관계=엣지로 변환하며, 온톨로지/스키마와 도메인 전문가가 변환을 안내한다.
- LLM은 방대한 비정형 텍스트를 고차원 벡터 공간으로 인코딩하며 대부분 비지도(unsupervised)로 학습해 전처리 부담이 적지만 지식이 불투명하다.
- 접근성(access): LLM은 수십억 파라미터의 암묵적(implicit) 표현으로 불투명하고, KG는 노드·관계·속성의 명시적(explicit) 표현으로 해석 가능하다.
- 갱신(updates): KG는 노드·관계 추가/수정/삭제로 갱신하나, LLM은 재훈련·파인튜닝이 필요해 훨씬 복잡하다.
- 능력(capabilities): LLM은 자연어 이해·생성에 강하고, KG는 개발자의 접근 패턴·스키마 설계에 크게 의존한다.
- 두 방식은 보완적이며, 하이브리드 접근은 형식적 추론과 확률적·맥락적·패턴 기반 계산을 결합해 더 넓은 작업 스펙트럼을 다룬다.

## 4. Reasoning

> 추론 엔진은 사용자 요청(목표·정보)을 받아 통찰과 제안을 제공하며, 불확실성·추론(inference)·추상화라는 열린 질문을 다룬다.

- 스팸 필터 예시: 단순 암기·매칭은 일반화 능력이 없어 진정한 학습이 아니며, 보지 못한 사례를 분류하는 능력이 귀납 추론(inductive inference)이다.
- 연역 추론은 일반 전제에서 논리적으로 참인 구체 결론을 도출하나(전제가 참이면 결론도 참), 귀납 추론은 구체 관찰에서 일반화하여 거짓 결론을 낳을 수 있다.
- LLM의 "추론" 한계: 농부와 양 강 건너기 문제(늑대·양배추 없는 단순화 버전)에서 LLM은 훈련 데이터의 전체 문제와 확률적으로 가까운 답을 내놓아 비효율적 왕복을 제안했다.
- Wu et al.의 11개 과제 연구: 기본 과제에서 벗어난 반사실적(counterfactual) 변형에서 성능이 일관되게 크게 저하되어, LLM이 좁고 전이 불가능한 절차에 의존함을 보여준다.
- 결론: KG는 정밀·규칙 기반 추론에, LLM은 패턴 인식·맥락 이해·모호성 처리에 쓰되, 둘 다 인간 수준의 상식 추론(common-sense reasoning)은 없다.

---

## 5. Reasoning Engines

> 추론 엔진은 지식 베이스를 읽을 뿐 아니라 다시 기록하며, 행동→환경→관찰→새 지식의 반복적 피드백 루프를 형성한다.

- 순수 연역 추론의 한계: 비용·결과·확률 관계를 담은 완전하고 정확한 지식 베이스가 필요하나 현실에서는 드물다.
- 귀납 추론과 ML: 관련 온톨로지·관계를 학습해 지식 베이스를 확장하고, 불확실성 하에서 불완전 데이터로부터 일반화·예측한다.
- LLM의 역할: 핵심 정보가 빠진 상황에서도 확률적 추론으로 맥락 적합한 제안을 생성하며(예: 비특이적 증상의 다중 진단 가능성 평가), KG 기반 엔진과 통합 시 모호한 입력을 해석하는 추론 계층이 된다.

```
  Knowledge Base <----writes back---- Reasoning Engine
        |                                    ^
     reads                                   |
        v              actions/suggestions   |
   Reasoning Engine ----> Environment ----> Observations
   (feedback loop: continuous improvement)
```

## 6. A KG Approach to IASs

> KG는 IAS 개발의 모든 곳에 적용되며, 본서는 가용 데이터가 아닌 비즈니스 목표에서 출발하는 목적 주도 구축을 권장한다.

- 상향식(bottom-up) 접근: 모든 데이터를 먼저 단일 진실 원천으로 통합한 뒤 발견을 시작하나, 소스가 너무 많고 정규화 부담이 커 KG 도입 실패로 이어지기 쉽다.
- CRISP-DM을 KG 플랫폼에 재해석: 비즈니스 이해→데이터 이해→KG 콘텐츠·구조 정의→모델링→평가→배포의 목적 주도 프로세스에서 KG가 중심 지식 표현이 된다.
- 획득 단계에서 LLM이 비정형 데이터에서 엔티티·관계를 추출하고 감정 분석·주제 식별 같은 일반 이해를 제공한다.
- 스키마리스(schemaless) 그래프는 기존 데이터·기능을 해치지 않고 새 노드·관계 타입으로 확장 가능해, 2차 라운드부터는 빈 KG에서 시작하지 않는다.
- 예측 모델(predictive model)은 학습 결과를 효율적으로 표현한 공식으로, 미지의 타깃 값을 추정하는 데 사용된다.

## Summary (핵심 정리)

- 지능은 지식의 획득·적용이며 지식 표현과 추론이 지능형 시스템 아키텍처의 핵심 구성요소로, KG는 명시적·해석가능 표현을, LLM은 암묵적·언어이해 표현을 담당한다.
- 시스템은 자율형과 조언형(IAS)으로 나뉘고, KG와 LLM을 결합한 하이브리드가 정밀 추론과 모호성·맥락 처리라는 상호 보완 강점을 활용한다.
- 가용 데이터 중심 상향식보다 비즈니스 목표에서 출발하는 목적 주도 KG 구축이 훨씬 효과적이며 프로젝트 실패를 막는다.
