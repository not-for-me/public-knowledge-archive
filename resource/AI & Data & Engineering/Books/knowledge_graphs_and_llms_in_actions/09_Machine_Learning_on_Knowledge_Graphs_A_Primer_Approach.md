# 09. Machine Learning on Knowledge Graphs: A Primer Approach

## 챕터 개요 (3줄 요약)

- 그래프는 보편적 데이터 표현을 제공하고 다양한 문제를 소수의 계산 과제로 모델링하며, 데이터 항목 간 의존성을 자연스럽게 담아 ML에 가치를 더한다.
- 그래프 ML 과제는 노드 중심(노드 분류·링크 예측·커뮤니티 탐지)과 그래프 중심(그래프 분류)으로 나뉘며, 전통 ML의 i.i.d. 가정을 깬다.
- 핵심은 피처 엔지니어링(featurization)으로, 노드를 벡터로 표현하는 방식이 성공을 좌우하며 도메인 지식(동질성 homophily) 활용이 단순 지표보다 우수하다.

---

## 1. Machine Learning on Graphs: Why?

> (지식)그래프에 ML을 수행하는 것이 가치 있고 때로는 유일한 선택인 이유가 있다.

- 데이터 표현: 행렬·텐서·시퀀스·시계열 등 다양한 데이터를 그래프라는 보편적 표현으로 변환할 수 있다.
- 문제 모델링: 이상 노드 탐지·약물 추천은 노드 분류로, 추천·상호작용 식별은 관계 예측으로 환원되는 소수 계산 과제로 다룰 수 있다.
- 데이터 항목 의존성: 전통 ML은 데이터가 독립·동일 분포(i.i.d.)라 가정하나, 현실은 연결되어 있어 관계를 무시하면 잘못된 결과를 낳는다.
- GNN(Graph Neural Network)과 LLM의 결합은 GNN의 구조적 패턴 탐지와 LLM의 자연어 이해·생성을 합쳐 강력한 IAS를 만든다(예: 정밀의료에서 환자 유사성 네트워크 처리 + 의학 문헌 종합 설명).

---

## 2. Machine Learning on Graphs: What?

> 그래프 ML 과제는 노드 중심과 그래프 중심으로 나뉘며, 지도/비지도 구분이 깔끔히 맞지 않는다.

### 9.2.1 Node Classification

- 라벨이 일부만 있는 노드의 라벨을 예측하는 과제(예: 소셜 네트워크의 봇 탐지, 단백질 기능 분류, 문서 주제 분류).
- 노드는 i.i.d.가 아니므로(동질성 homophily, 구조적 등가성, 이질성 heterophily) 노드 특성과 네트워크 관계를 함께 모델링해야 한다.
- 전체 그래프(미라벨 노드의 이웃 정보 포함)에 접근하므로 준지도(semisupervised)로 간주되나, 표준 준지도가 요구하는 i.i.d.를 만족하지 않아 분류가 까다롭다.
- 다중 라벨 노드 분류로 확장 가능하다(예: Flickr 사용자의 관심 그룹 예측).

### 9.2.2 Link Prediction (Relationship Prediction)

- 노드 간 미래·누락 연결을 식별하는 과제(예: 공저 그래프의 향후 협업 예측, 친구·제품 추천, KG 완성, 약물 부작용 예측, 단백질 상호작용 발견).
- 그래프는 미관측·은닉 연결이나 진화 특성 때문에 불완전하며, 누락 엣지 추론이 많은 IAS에 도움된다.
- 링크 예측은 연결 존재만, 관계 예측(relationship prediction)은 연결의 타입까지 다루며, 역시 준지도로 접근된다.

### 9.2.3 Clustering and Community Detection

- 네트워크 토폴로지만으로 잠재 그룹 구조를 발견하는 과제(예: 유전 상호작용 네트워크의 기능 모듈, 금융 거래의 사기 그룹).
- 대규모 그래프의 구조적 요약을 제공하며, K-means·DBSCAN과 달리 독립 점이 아닌 상호 연결 데이터를 다룬다.
- 일반적으로 비지도이나 label propagation처럼 기존 라벨을 활용하는 접근도 있다.

### 9.2.4 Graph Classification

- 다수의 독립 그래프를 다루며 각 그래프가 라벨을 가진 i.i.d. 데이터 점이다(예: 분자의 독성·용해도 예측 — 원자=노드, 결합=엣지).
- 그래프 전체를 라벨에 매핑하는 학습으로, 효소 분류·악성코드 분류 등에 응용되며 그래프 내부 구조·구성요소 특성 포착이 핵심 도전이다.

```
  Node-focused:  one graph, data points = nodes/edges
                 (node classification, link prediction, community detection)
  Graph-focused: set of graphs, data point = whole graph
                 (graph classification)
```

## 3. Machine Learning on Graphs: How?

> 구현은 그래프 전용 알고리즘(collective classification)과 그래프를 피처 벡터로 변환해 전통 ML을 쓰는 두 방향이 있다.

- 본 파트는 그래프 피처 엔지니어링에 집중하며, 수동 추출→반자동 접근→GNN(자동 피처 학습)으로 진행한다.
- GNN은 구조적 패턴과 노드 속성을 모두 포착해 불완전한 KG에 특히 유용하며, 다운스트림 분류에 도움되는 벡터 임베딩을 만든다.

### 9.3.1 Node Classification and Link Prediction

- 학습·예측 모두 노드·관계의 featurization이 핵심 단계이며, 학습과 예측의 featurization은 반드시 동일해야 한다.
- Zachary Karate Club 그래프(34명, 분열 후 두 클럽 소속)로 노드 분류를 시연한다.
- 단순 임베딩(노드 차수)은 f-score가 30~70%로 불안정해, 한계는 분류기가 아닌 피처 엔지니어링에 있음을 보인다.
- Node2Vec(랜덤 워크 기반 표현 학습)도 신중한 튜닝이 필요하며 단독으로는 최적이 아니다.
- 피처 스케일링(StandardScaler)은 차수(1~1000)와 중심성(0~1)처럼 다른 척도의 피처가 균등 기여하게 해 유클리드 거리 기반 알고리즘의 편향을 막는다.
- 로지스틱 회귀(logistic regression)는 이름과 달리 분류 알고리즘으로, 선형 결합을 0~1 확률로 변환한다.
- 동질성 활용: 노드를 (전체 차수, Mr. Hi 이웃 차수, Officer 이웃 차수) 3원소 벡터로 표현하니 f-score가 약 93%로 안정적으로 향상되었다.

---

## 9.3.2 ~ 9.3.3 Graph Classification & Clustering

> 그래프 분류는 노드 대신 그래프 전체의 피처를 추출하며, 그래프 클러스터링은 변환·학습 없이 그래프를 직접 다룬다.

- 그래프 분류도 featurization이 핵심이며 출력 품질은 피처 품질·알고리즘 튜닝에 크게 의존한다.
- 그래프 클러스터링은 피처 추출이 없고(노드 관계·방향·가중치 직접 사용), WCC는 고정 메커니즘, Louvain·LPA(Label Propagation Algorithm)는 모듈성 등을 최적화한다.
- LPA는 라벨을 네트워크에 전파하며 빠르나 무작위성으로 실행마다 결과가 약간 다르다.
- Karate Club에 Louvain을 적용하니 4개 커뮤니티를 식별했고, 하나의 가짜(spurious) 노드(8번)만 다른 그룹과 잘 연결되어 있었다 — 학습 단계가 없는 비지도 접근이다.

---

## Summary (핵심 정리)

- 그래프 ML은 상호 연결 데이터를 자연스럽게 다루고 보편적 표현을 제공하며, 전통 ML의 i.i.d. 가정 대신 노드 간 연결·의존성을 활용해 실세계를 더 잘 반영한다.
- 과제는 노드 중심(노드 분류·링크 예측, 보통 준지도)과 그래프 중심(그래프 분류)으로 나뉘고, 커뮤니티 탐지는 보통 비지도로 그래프 구조에 직접 작동한다.
- 피처 엔지니어링이 결정적이며, 노드 임베딩은 수동부터 Node2Vec까지 다양하나 신중한 튜닝과 도메인 지식(동질성) 결합이 성공의 균형을 좌우한다.
