# 01. Knowledge Graphs and LLMs: A Killer Combination

## 챕터 개요 (3줄 요약)

- 생성형 AI는 강력하지만 도메인 지식, 정확성, 설명가능성이 필요한 영역에서 환각(hallucination)과 맥락 부재라는 한계를 드러낸다.
- 지식 그래프(KG, Knowledge Graph)는 검증된 구조적 지식을 제공하고, LLM(Large Language Model)은 비정형 텍스트에서 엔티티/관계 추출과 자연어 인터페이스를 제공하여 상호 보완한다.
- 두 기술의 결합은 이질적 데이터를 통합하고 자연어로 질의 가능한 "신뢰의 단일 원천(single source of truth)"을 만드는 패러다임 전환을 이끈다.

---

## 1. Knowledge Graphs

> KG는 인간의 지식을 노드(entity)·관계(relationship)·속성(property)이라는 정교한 그래프 구조로 기계에 인코딩하여 추론과 추리를 가능하게 한다.

- 노드는 사람·장소·질병·단백질·유전자 같은 실세계 엔티티를 나타내고, 관계는 그들 사이의 의미 있는 연결(예: 사람이 태어난 장소, 유전자가 인코딩하는 단백질)을 정의한다.
- 속성(property)은 출생일·지리 좌표·질병 설명 등 맥락 정보를 부여하여 기계가 구조화된 지식 위에서 추론·추리(reasoning and inference)를 수행하게 한다.
- KG는 효과적임에도 구축·유지 비용이 크고, 멀티홉(multi-hop) 탐색을 위한 복잡한 접근 패턴이 필요하며, 결과가 여러 노드·관계에 흩어져 있어 널리 채택되지 못했다.
- 구조화/반구조화 데이터(RDB, CSV, XML, JSON)는 항목이 분리·식별·타입화되어 공통 그래프 스키마로의 매핑이 통제 가능하고 예측 가능하다.
- 비정형 텍스트 추출은 다국어, 오타, 대명사 참조 해소(coreference resolution), 다양한 문체, 도메인 전문 용어 때문에 본질적으로 어렵다.
- 유연한 스키마는 이질적 데이터 처리에 유리하지만 질의 방법을 모르는 사용자에게는 접근을 어렵게 하고, 결과 해석도 비전문가에게 난해하다.
- Google이 초기 도입자로 "문자열이 아닌 사물(things, not strings)"을 검색하도록 KG를 활용했으나 검색 애플리케이션에 국한되었다.

---

## 2. Large Language Models

> LLM은 자연어 처리를 전담하여 KG 기반 지능형 시스템의 진화를 가로막던 장벽을 제거한다.

- LLM의 기반은 전이 학습(transfer learning)으로, 마스킹된 토큰 예측 같은 일반 작업에서 학습한 패턴을 관계 추출 같은 특정 작업에 재사용한다.
- 이는 작고 작업별인 다수 모델 대신 여러 작업에 재사용 가능한 소수의 대형 모델을 훈련하는 패러다임 전환을 일으켜 학습 데이터와 연산 자원을 크게 줄였다.
- 트랜스포머(transformer) 아키텍처로 대규모 코퍼스에 사전학습된 PLM(Pretrained Language Model)은 하나의 큰 모델로 다양한 NLP(Natural Language Processing) 작업을 수행했다.
- LLM은 수백억~수천억 개 파라미터를 가진 대규모 PLM을 의미하며 GPT-2, GPT-4, Gopher, PaLM 등이 대표적이다.
- 성능이 "비합리적으로(unreasonably)" 뛰어난 이유는 모델 복잡도(파라미터 수), 학습 코퍼스의 규모·품질, 인간 지능이 필요한 작업을 다음 토큰 예측(next token prediction)으로 환원하는 능력이다.
- "Scaling Laws" 연구에 따르면 더 큰 모델은 동일 성능 도달에 더 적은 데이터 샘플을 요구하며, 코퍼스 규모와 데이터 품질이 모두 결정적이다.
- 전통적 모델 중심(model-centric) 패러다임에서 데이터 엔지니어링을 우선하는 데이터 중심(data-centric) 패러다임으로의 전환이 핵심이다.

## 3. KGs and LLMs: Stronger Together

> KG와 LLM은 서로의 한계를 보완하며 강력한 지능형 시스템 구현을 강화한다.

- LLM은 KG 구축(비정형 데이터에서 개념·연결 추출), KG 질의(스키마/질의언어 이해를 도와 멀티홉 탐색 지원), 요약(KG 결과를 텍스트 형태로 반환)에서 KG를 지원한다.
- 과거 도메인별 맞춤 NLP 모델 훈련이 필요하던 작업을 LLM은 프롬프트만으로 다목적 처리할 수 있게 단순화했다.
- KG는 LLM의 환각을 검증된 구조적 지식이라는 사실 기반으로 줄이고, text-to-cypher 변환으로 자연어 질문을 정밀한 그래프 질의로 변환한다.
- KG는 GraphRAG와 KG 기반 프롬프팅으로 지속 업데이트되어 LLM의 정보 노후화(stale information) 문제를 완화한다.
- GraphRAG는 지식을 의미 있는 클러스터와 커뮤니티 요약으로 조직하여 최신 정보를 LLM에 제공한다.
- KG는 추적·검증 가능한 정보 경로와 구조적 추론을 제공해 설명가능성(explainability)을 높인다.

## 4. The Paradigm Shift in Data-Driven Applications

> KG는 연결을 포착하여 그래프 패턴 매칭과 순회로 관계 발견을 가능하게 하며, 지능형 행위를 단일 진실 원천에 인코딩하는 패러다임 전환을 이끈다.

- 전통적 패러다임은 구조화·동질적 DB로 특정 목적 시스템을 만들지만 복잡 도메인과 이질적 데이터 통합에는 부적합하다.
- RDF(Resource Description Framework)와 LPG(Labeled Property Graph)는 모두 인간이 해석 가능한 기계 판독 형식을 제공한다.
- McKinsey에 따르면 데이터 파편화 해결로 단기 연간 데이터 지출을 5~15% 절감할 수 있다.

### 1.4.1 The Four Pillars of Knowledge Graphs

- KG의 새 정의: 특정 도메인을 위해 구축되고 구조화·비정형 데이터를 통합하는, 타입화된 엔티티·속성·의미 있는 명명된 관계로 구성된 끊임없이 진화하는 그래프 데이터 구조.
- 네 가지 기둥(four pillars): 진화(Evolution), 의미(Semantics), 통합(Integration), 학습(Learning).
- 진화는 전면 개편 없이 새 정보를 단일 원천으로 지속 통합하고, 의미는 타입 엔티티·관계로 데이터 의미를 명시화하여 설명가능성과 추론(일관성 검사~인과 추론)의 근간이 된다.
- 통합은 데이터 타입·형식·출처 문제를 넘어 다중 소스를 연결하는 중심 참조 역할을 하고, 학습은 중심성·연결성·네트워크·커뮤니티 분석과 ML로 명시되지 않은 새 정보를 추론한다.

```
        KNOWLEDGE GRAPH
   +---------------------------+
   | Evolution | Semantics     |
   |-----------|---------------|
   | Integration | Learning    |
   +---------------------------+
   (four pillars: ever-evolving single source of truth)
```

## 5. Building Data-Driven Applications Using KGs and LLMs

> 헬스케어는 데이터 풍부성과 적용 가능성 때문에 본서의 주요 예시 도메인으로 쓰인다.

- 신약 발견(drug discovery): KG가 유전자·질병·약물을 타입 관계로 연결하고, LLM이 논문·임상 보고서의 비정형 데이터를 처리·통합하여 가설 생성과 텍스트 마이닝을 가속한다.
- 고객 지원 대화형 AI: 대화는 지식 주변에서 전개되므로 KG가 개념을 연결하여 대화를 그라운딩(grounding)하고, LLM이 일관된 응답을 생성한다.
- LLM은 추가 맥락 그라운딩 없이는 응답이 일반적이고 깊이가 부족해질 수 있어 KG와의 통합이 정확성과 맥락성을 높인다.
- KG 채택 판단: 데이터 사일로 통합, 구조·비정형 소스 연결, 진화하는 유연한 표현, 파이프라인 출처 추적, 검색·추천, 네트워크 시각화, 관계형 ML 필요 시 KG가 적합하다.
- LLM 채택 판단: 비정형 데이터에서 엔티티·관계 추출, 복잡 질의 해석, 대화형 인터페이스, 결과의 텍스트 요약이 필요할 때 LLM으로 KG 솔루션을 강화한다.

## 6. Knowledge Graph Technologies

> 본서는 기술 중립적 접근으로 KG 생성·질의의 두 주요 패러다임인 RDF/SPARQL과 LPG/openCypher·Gremlin을 교차 사용한다.

- RDF는 W3C가 정의한 데이터 모델로 그래프를 진술(statement)·트리플(triplet)의 집합으로 인코딩하며 웹 데이터 게시·공유 표준화와 의미 계층 추론에 초점을 둔다.
- LPG는 노드·엣지가 속성을 갖는 구조에 초점을 두며 openCypher, Gremlin으로 질의한다.
- RDF는 표준화된 진술을 통한 데이터 상호운용성·일관성과 하이퍼그래프·페더레이션에 강하다.
- LPG는 경로 탐색(pathfinding) 질의와 그래프 순회에 강하며, 각 엣지가 고유 정체성·속성을 가진다(RDF에서는 관계가 재사용 가능한 전역 술어).
- 두 패러다임은 용도에 따라 상호 보완적이다.

### 1.6.1 Taxonomies and Ontologies

- 분류체계(taxonomy)는 데이터의 계층적 차원을 넓음-좁음(broader–narrower) 관계로 조직한다(예: Vehicle > Car > Sedan).
- 온톨로지(ontology)는 단순 계층을 넘어 동일성·차이·합집합·여집합·서로소(disjoint)·카디널리티 제약 등 복잡한 관계로 도메인 개념 구조를 포착한다.
- 현대 KG는 "충분한 만큼의 의미(just enough semantics)"라는 실용적 접근으로 현재 문제를 다루는 온톨로지 기능 부분집합을 선택하고 유기적으로 확장한다.

---

## Summary (핵심 정리)

- KG는 타입 엔티티·속성·의미 있는 관계를 담은 끊임없이 진화하는 그래프 구조이며, 진화·의미·통합·학습이라는 네 기둥으로 정의된다.
- LLM과 KG는 서로의 한계(환각, 정보 노후화, 설명 불가 vs 구축 비용, 질의 난이도)를 보완하며, 지능형 행위를 단일 신뢰 원천에 인코딩하는 패러다임 전환을 만든다.
- KG의 두 핵심 기술은 RDF와 LPG이며, 분류체계·온톨로지가 의미 메타데이터를 더해 전통 그래프를 지능화한다.
