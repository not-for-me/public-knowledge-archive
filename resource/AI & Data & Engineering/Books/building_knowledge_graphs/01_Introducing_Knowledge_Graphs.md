# 1. Introducing Knowledge Graphs

## 챕터 개요 (3줄 요약)
- 지식그래프(Knowledge Graph)는 노드(Node)와 관계(Relationship)로 도메인을 고충실도로 표현하고, 데이터에 "맥락(Context)"을 입혀 비즈니스 가치를 끌어내는 기술이다.
- 이 책은 LPG(Labeled Property Graph, 레이블 속성 그래프) 모델을 기반으로 하며, 단순한 원시 요소(노드/레이블/관계/속성)로 복잡한 현실을 모델링한다.
- 지식그래프의 핵심은 데이터 위에 "조직 원리(Organizing Principle)"라는 의미 계층을 두어, 데이터 자체가 똑똑해지고 추론(Reasoning)이 가능해진다는 점이다.

## What Are Graphs?
> 그래프는 노드(정점)를 관계(간선)로 연결해 도메인을 표현하는 단순하지만 강력한 구조이며, 차트(chart)와는 전혀 다른 개념이다.

- 그래프 이론은 18세기 오일러(Euler)의 쾨니히스베르크 다리 문제에서 출발했으며, 현실의 노이즈를 걷어내고 "연결 관계"만 논리적으로 남기는 것이 핵심이다.
- 이는 소프트웨어 개발과 유사하다: 현실을 추상화한 논리적 표현만 남긴다.
- 관계는 방향(directed) 유무, 다중 노드 연결(hypergraph) 등 여러 모델이 존재하나, 이 책은 실무 친화적인 LPG 모델을 채택한다.
- LPG는 수학적 그래프보다 정보가 풍부(information rich)해 복잡한 도메인 표현에 적합하다.

### The Property Graph Model
> 속성 그래프 모델은 노드, 레이블, 관계, 속성이라는 네 가지 원시 요소로 구성된 현대 그래프 DB의 표준 모델이다.

- 노드(Node): 도메인의 엔터티를 표현하며, 0개 이상의 속성(key-value)과 0개 이상의 레이블(예: Customer, Product)을 가진다.
- 관계(Relationship): 타입(BOUGHT, FOLLOWS 등), 방향, 선택적 속성을 가지며 절대 "dangle"(시작/끝 노드 없이 떠있음)하지 않는다.
- 스키마는 "schema-ish": 필요한 곳에만 제약(constraint)을 걸고, 전체를 미리 고정하지 않아 현실의 불균형/불완전한 데이터를 수용한다.
- 노드/관계마다 속성 수가 달라도 정상이며, 모델이 문제 도메인과 일치하는 것만이 중요하다.

```
(Person {name:'Rosa'}) -[:LIVES_IN {since:2020}]-> (Place {city:'Berlin'})
(Person {name:'Rosa'}) -[:FRIEND]-> (Person {name:'Karl'})
(Person {name:'Karl'}) -[:FRIEND]-> (Person {name:'Fred'})
```

> [모델링 관점 - 주식시장 도메인 적용]
> LPG의 "schema-ish" 유연성은 주식시장 지식그래프에 결정적이다. 예: (Company)-[:LISTED_ON]->(Exchange), (Company)-[:HAS_SUBSIDIARY]->(Company), (Investor)-[:HOLDS {shares, since}]->(Company), (Company)-[:SUPPLIES]->(Company) 처럼 기업/투자자/거래소/이벤트를 노드로, 지분/공급/경쟁/규제를 관계로 모델링하면 재무제표만으로는 안 보이는 "맥락적 연결"이 드러난다. 관계 속성(지분율, 체결시각, 거래량)을 적극 활용하는 것이 시장 데이터 모델링의 핵심이다.

## The Motivation for Knowledge Graphs
> 데이터 폭증과 빠른 의사결정 수요 속에서, 맥락화된 데이터로 지식을 생성하려는 강한 수요가 지식그래프를 부상시켰다.

- 팬데믹·지정학 등 외부 충격으로 의사결정 속도 요구가 급증했고, 과거 지식은 빠르게 무효화된다.
- 고객경험, 사기탐지(fraud detection), 자동화, 예측 개선 등 광범위한 영역에서 맥락 데이터가 필요하다.

## Knowledge Graphs: A Definition
> 지식그래프는 현실의 엔터티·사건·관계를 인간과 기계가 모두 이해 가능한 형태로 표현한, 상호 연결된 사실(fact)의 집합이다.

- 모든 그래프가 지식그래프는 아니다 — 핵심 차별점은 "조직 원리(Organizing Principle)"라는 의미 계층의 존재이다.
- 조직 원리는 데이터에 구조와 맥락을 더해 추론을 가능케 하고, "똑똑한 애플리케이션 + 멍청한 데이터"라는 통념을 뒤집어 데이터 자체를 똑똑하게 만든다.
- 물리적 저장 방식에는 불가지론적(agnostic): 단일 그래프 DB, 그래프 연합(federation), 데이터 레이크 위 구조화 계층, 가상화 인덱스~완전 구체화까지 모두 가능하다.

## Summary (핵심 정리)
- 지식그래프는 노드·관계·속성이라는 단순 원시 요소(LPG 모델) 위에 "조직 원리"라는 의미 계층을 더해 데이터에 맥락과 추론 능력을 부여하는 기술이다.
- 데이터 플랫폼/온톨로지 관점에서 가장 중요한 설계 결정은 "무엇을 노드로, 무엇을 관계로, 어떤 속성을 관계에 둘 것인가"이며, schema-ish 유연성이 현실 데이터의 불완전성을 수용한다.
- 주식시장 도메인에서는 기업·투자자·거래소·규제·이벤트를 연결해, 재무 수치만으로 보이지 않는 자본주의 사회의 구조적 맥락을 드러내는 것이 지식그래프의 가치 제안이다.
