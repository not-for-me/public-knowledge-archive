# 01. Introduction

## 챕터 개요 (3줄 요약)

- 소프트웨어 아키텍처는 정해진 정답이 없고, 시대적·기술적 맥락 속에서 형성되는 트레이드오프의 산물임을 강조한다.
- 아키텍처를 구성하는 네 가지 차원(아키텍처 스타일, 아키텍처 특성, 논리적 컴포넌트, 아키텍처 결정)과 세 가지 법칙을 소개한다.
- 소프트웨어 아키텍트에게 기대되는 8가지 핵심 역량과 책 전체의 구성(3부)을 안내한다.

---

## 1. Defining Software Architecture (소프트웨어 아키텍처의 정의)

> 소프트웨어 아키텍처는 아키텍처 스타일을 출발점으로, 아키텍처 특성·논리적 컴포넌트·아키텍처 결정이 결합된 4차원 구조로 정의된다.

저자들은 아키텍처를 네 가지 차원으로 본다. 첫째, 시스템이 지원해야 하는 아키텍처 특성("-ilities")은 시스템이 무엇을 해야 하는지에 대한 성공 기준이다. 둘째, 논리적 컴포넌트는 시스템의 행위(behavior)를 정의하며 도메인·엔티티·워크플로우를 형성한다. 둘을 분석한 뒤 적절한 아키텍처 스타일을 출발점으로 선택한다. 마지막으로 아키텍처 결정은 시스템 구축 규칙이자 제약으로, 개발팀에게 허용/비허용을 알려준다. 아키텍처 결정과 문서화는 21장에서 다룬다.

```
        +----------------------------------+
        |       Software Architecture       |
        +----------------------------------+
         |          |          |          |
     Style    Characteristics  Components  Decisions
   (starting   (-ilities,      (behavior:  (rules &
    point)      success)       domains)    constraints)
```

## 2. Laws of Software Architecture (소프트웨어 아키텍처 법칙)

> 보편적으로 성립하는 세 가지 법칙을 통해 아키텍처 사고의 핵심 관점을 제시한다.

제1법칙: 소프트웨어 아키텍처의 모든 것은 트레이드오프다. 트레이드오프가 없어 보인다면 아직 발견하지 못한 것일 뿐이며(따름정리1), 트레이드오프 분석은 한 번으로 끝나지 않는다(따름정리2). 제2법칙: "어떻게(how)"보다 "왜(why)"가 더 중요하다. 기존 아키텍처가 어떻게 동작하는지는 파악할 수 있어도, 왜 그런 결정을 했는지는 맥락 없이는 알기 어렵다. 제3법칙: 대부분의 아키텍처 결정은 이분법이 아니라 양극단 사이의 스펙트럼 위에 존재한다.

## 3. Expectations of an Architect (아키텍트에게 기대되는 것)

> 역할·직함과 무관하게 모든 아키텍트에게 공통적으로 기대되는 8가지 핵심 역량을 정리한다.

아키텍트는 아키텍처 결정을 내리되 기술을 '강제'하기보다 '안내(guide)'해야 한다. 또한 아키텍처를 지속적으로 분석(architecture vitality)하여 구조적 부패(structural decay)를 막고, 최신 기술·산업 트렌드를 따라가야 한다. 결정 준수 여부를 검증하고, 다양한 기술에 대한 기술적 폭(breadth)을 갖추며, 비즈니스 도메인을 이해해야 한다. 마지막으로 리더십·대인관계 기술과 조직 정치 탐색·협상 능력이 필수적이다. 소프트 스킬은 아키텍트 성공의 절반 이상을 차지한다.

### 8가지 기대 역량 요약

```
1. Make architecture decisions     5. Understand diverse technologies
2. Continually analyze architecture 6. Know the business domain
3. Keep current with trends         7. Possess interpersonal skills
4. Ensure compliance                8. Navigate politics
```

## 4. Roadmap (책의 구성)

> 본서는 기초, 아키텍처 스타일, 기법·소프트 스킬의 3부로 구성된다.

Part I(Foundations)은 아키텍처 특성과 논리적 컴포넌트라는 두 핵심 구조 요소를 분석하는 기법을 다룬다. Part II(Architecture Styles)는 명명된 토폴로지인 아키텍처 스타일들의 카탈로그를 제공하며 구조·통신 차이를 비교한다. Part III(Techniques and Soft Skills)는 기술 외에 사람과 관련된 소프트 스킬을 다루며, 이는 아키텍트에게 가장 습득하기 어렵지만 매우 중요하다.

---

## Summary (핵심 정리)

- 아키텍처는 맥락의 산물이며, 스타일·특성·컴포넌트·결정의 4차원으로 정의된다.
- 세 가지 법칙(트레이드오프, why>how, 스펙트럼)이 아키텍처 사고의 토대를 이룬다.
- 아키텍트는 8가지 역량(기술 결정·분석·트렌드·준수·기술 폭·도메인·대인관계·정치)을 갖춰야 한다.
